# AI 使用指南

## 数据查询最佳实践

### 1. 命令执行方式

所有数据查询必须通过 CLI 命令执行：

```bash
npx -y westock-data-clawhub@1.0.4 <command> <stock_code> [options]
```

### 2. 输出格式

CLI 命令输出为 Markdown 表格格式，AI 应直接从表格中读取和解析数据。

**示例输出格式**：

```
股票代码 | 股票名称 | 最新价 | 涨跌幅 | 成交量
-------- | -------- | ------ | ------ | ------
sh600000 | 浦发银行 | 10.23 | +1.23% | 12.5M
```

### 3. 数据解析指南

#### K线数据解析

```bash
npx -y westock-data-clawhub@1.0.4 kline sh600000 --period day --limit 20
```

返回字段：
- **date**: 交易日期
- **open**: 开盘价
- **high**: 最高价
- **low**: 最低价
- **close**: 收盘价
- **volume**: 成交量
- **amount**: 成交额
- **turnover**: 换手率

**数据分析示例**：
```
分析要点：
1. 成交量趋势：连续放量/缩量
2. 均线系统：MA5/MA10/MA20 排列
3. 涨跌幅度：日涨跌、周涨跌
4. 振幅：当日最高-最低的幅度
```

#### 资金流向解析

```bash
npx -y westock-data-clawhub@1.0.4 asfund sh600000
```

返回字段：
- **MainNetFlow**: 主力净流入（万元）
- **JumboNetFlow**: 大单净流入（万元）
- **MidNetFlow**: 中单净流入（万元）
- **SmallNetFlow**: 小单净流入（万元）
- **MainNetFlowRate**: 主力净流入占比（%）

**数据分析示例**：
```
资金流向分析：
- 主力净流入为正且持续 → 机构建仓信号
- 主力净流出持续 → 机构减仓信号
- 超大单和大单分化 → 注意主力对倒
```

#### 技术指标解析

```bash
npx -y westock-data-clawhub@1.0.4 technical sh600000 --group macd,rsi
```

返回字段（MACD）：
- **DIF**: 快线
- **DEA**: 慢线
- **MACD**: 柱状值（DIF-DEA）

返回字段（RSI）：
- **RSI6**: 短期RSI（6日）
- **RSI12**: 中期RSI（12日）
- **RSI24**: 长期RSI（24日）

**技术分析示例**：
```
MACD 分析：
- DIF 上穿 DEA（金叉）+ 柱状由负转正 → 买入信号
- DIF 下穿 DEA（死叉）+ 柱状由正转负 → 卖出信号
- DIF 和 DEA 在零轴上方 → 强势市场

RSI 分析：
- RSI < 30 → 超卖区域，可能反弹
- RSI > 70 → 超买区域，可能回调
- RSI 背离 → 价格创新低但 RSI 未创新低 → 底部信号
```

### 4. 批量查询处理

#### 单股查询
```bash
npx -y westock-data-clawhub@1.0.4 kline sh600000 --period day --limit 20
```

#### 批量查询
```bash
npx -y westock-data-clawhub@1.0.4 kline sh600000,sh600519,sh600036 --period day --limit 20
```

**批量数据处理**：
```
处理步骤：
1. 逐个解析每个股票的返回数据
2. 按股票代码分组展示
3. 对比分析：同行业股票对比、资金流向对比
4. 提取共性和差异
```

### 5. 时间范围查询

#### 历史区间查询
```bash
npx -y westock-data-clawhub@1.0.4 technical sh600000 --group macd --start 2026-02-01 --end 2026-03-01
```

**数据聚合分析**：
```
分析维度：
- 区间涨跌幅：起止价格对比
- 振幅统计：最大涨幅/跌幅
- 成交量趋势：区间总量、平均日量
- 均线多头/空头排列
- 资金净流入总额
```

### 7. 数据展示规范

#### ✅ 正确做法
- 将查询结果转换为 Markdown 表格
- 标注货币单位（港元/美元/人民币）
- 提供数据解读和分析
- 使用百分比格式展示涨跌

#### ❌ 错误做法
- 直接输出原始 JSON
- 不标注货币单位
- 将数据分析逻辑写成独立脚本
- 创建临时文件处理数据

### 8. 常见分析场景模板

#### 场景1：趋势分析
```
分析步骤：
1. 查询日K线数据（20日）
2. 计算涨跌幅和振幅
3. 分析均线排列（MA5/MA10/MA20）
4. 观察成交量变化趋势
5. 总结：上升趋势/下降趋势/震荡整理
```

#### 场景2：资金分析
```
分析步骤：
1. 查询资金流向数据
2. 计算主力净流入/净流出
3. 分析超大单、大单、中单、小单分布
4. 对比连续多日资金数据
5. 总结：主力建仓/减仓/观望
```

#### 场景3：技术指标分析
```
分析步骤：
1. 查询 MACD、RSI、KDJ 等指标
2. 识别金叉/死叉信号
3. 判断超买/超卖区域
4. 寻找背离信号
5. 综合给出技术面结论
```

#### 场景4：基本面筛选
```
分析步骤：
1. 查询公司简况（行业、主营业务）
2. 查询最新财务数据（利润、营收）
3. 查询分红历史
4. 对比同行业公司
5. 综合给出基本面评价
```

### 9. 数据质量判断

#### 数据完整性检查
- 检查是否返回了所有请求的字段
- 验证数据条数是否符合 limit 参数
- 检查日期连续性（是否有缺失交易日）

#### 数据合理性检查
- 涨跌幅是否在合理范围内（-10%~+10%为常见涨跌停限制）
- 成交量是否为正数
- 股价是否为正数
- 技术指标值是否在合理范围

#### 数据时效性检查
- 最新数据日期是否为最近交易日
- 历史数据是否连续无断层
- 注意数据延迟（实时行情可能有15分钟延迟）

### 10. 错误处理

#### 常见错误及处理

**错误1：股票代码不存在**
```
症状：返回空数据或"股票不存在"
处理：
1. 确认股票代码格式正确
2. 尝试使用 search 命令搜索正确代码
3. 确认市场类型（sh/sz/hk/us）
```

**错误2：查询时间段无数据**
```
症状：返回空数据
处理：
1. 确认日期格式正确（YYYY-MM-DD）
2. 确认日期为交易日（非节假日）
3. 缩短时间范围重试
```

**错误3：不支持的命令或参数**
```
症状：返回参数错误
处理：
1. 查阅 SKILL.md 确认命令和参数格式
2. 检查参数拼写是否正确
3. 确认该市场是否支持该功能（如港股不支持龙虎榜）
```

### 11. 性能优化建议

#### 批量查询优化
- 优先使用批量查询（逗号分隔）
- 避免逐个查询多个股票
- 合理设置 limit 参数（避免一次查询过多数据）

#### 数据缓存策略
- 对于不频繁变化的数据（如公司简况），可适当缓存
- 实时行情数据每次查询最新
- K线数据根据分析周期选择合适的数据源

#### 并发查询
- 可同时发起多个不同类型的查询
- 例如：同时查询K线和资金流向
- 注意总查询数量控制

### 12. 合规使用

#### 数据使用规范
- 所有数据仅供展示和分析
- 不得将数据用于商业营利
- 不得对数据进行二次分发
- 尊重数据版权和来源标注

#### 风险提示
- 市场有风险，投资需谨慎
- 数据仅供参考，不构成投资建议
- 投资决策需结合自身判断
- 必要时咨询专业人士
