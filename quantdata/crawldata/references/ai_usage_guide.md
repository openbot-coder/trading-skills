# AI 使用指南

## 数据查询最佳实践

### 1. 命令执行方式

所有数据查询必须通过 CLI 命令执行：

```bash
cd quantdata/crawldata/scripts
python get_quote.py <codes> [--detail] [--json]
python crawldata.py quote <codes> [--detail] [--json]
```

### 2. 输出格式

CLI 命令输出为 Markdown 表格格式，AI 应直接从表格中读取和解析数据。

**简单输出格式**：
```
========================================================================================================================

------------------------------------------------------------------------------------------------------------------------
sh000001     上证指数        4170.75    +10.58     +0.25%     4168.45    4172.27    4163.14    445329490    12:05:00
hf_GC         纽约黄金         2315.60    +12.50     +0.54%     2303.10    2318.90    2299.50    125468       12:05:00
========================================================================================================================
```

### 3. 数据解析指南

#### 股票/指数行情解析

```bash
python get_quote.py sh000001 sh000300 sz000001
```

返回字段（股票）：
- **code**: 完整代码
- **name**: 股票/指数名称
- **last_price**: 最新价
- **pre_close**: 昨收价
- **open**: 开盘价
- **high**: 最高价
- **low**: 最低价
- **change**: 涨跌额
- **change_pct**: 涨跌幅(%)
- **volume**: 成交量（手）
- **turnover**: 成交额（元）
- **turnover_rate**: 换手率(%)
- **date**: 交易日期
- **time**: 交易时间

**数据分析示例**：
```
股票分析要点：
1. 涨跌幅度：日涨跌、量价配合
2. 换手率：0-3%温和，3-5%活跃，>5%高换手
3. 振幅：(最高-最低)/昨收
4. 成交金额：评估流动性
```

#### 期货行情解析

```bash
python get_quote.py hf_GC hf_CL hf_SI
```

返回字段（期货）：
- **code**: 完整代码
- **name**: 期货品种名称
- **last_price**: 最新价
- **pre_close**: 昨收价
- **open**: 开盘价
- **high**: 最高价
- **low**: 最低价
- **change**: 涨跌额
- **change_pct**: 涨跌幅(%)
- **volume**: 成交量（手）
- **date**: 交易日期
- **time**: 交易时间

**期货分析示例**：
```
期货分析要点：
1. 趋势判断：短期/中期/长期趋势
2. 价格位置：相对高低位
3. 波动幅度：日振幅、周振幅
4. 成交量配合：放量上涨/下跌
5. 多品种对比：黄金-白银、原油-天然气
```

#### 买卖盘解析（--detail 模式）

```bash
python get_quote.py sh000001 --detail
```

**买卖盘解读**：
```
买盘分析：
- 买一价和买一量：短期支撑位
- 买五档深度：判断下方承接力
- 大单挂单：机构意图

卖盘分析：
- 卖一价和卖一量：短期压力位
- 卖五档深度：判断上方抛压
- 大单压单：主力意图
```

### 4. 批量查询处理

#### 单品种查询
```bash
python get_quote.py sh000001
python get_quote.py hf_GC --detail
```

#### 批量查询
```bash
python get_quote.py sh000001,hf_GC,hf_CL,sz000001
```

**批量数据处理**：
```
处理步骤：
1. 逐个解析每个品种的返回数据
2. 按品种类型分组展示（股票/指数/期货）
3. 对比分析：同类别品种对比、涨跌排名
4. 提取共性和差异
```

### 5. 时间处理

#### 实时数据
crawldata 提供实时行情数据，注意：
- 期货数据为全球时区
- A股交易时间：9:30-11:30, 13:00-15:00
- 非交易时间可能显示最后收盘数据

#### 时间格式
- 日期：YYYYMMDD（如 20260507）
- 时间：HH:MM:SS（如 12:05:00）
- 完整时间：YYYYMMDD HH:MM:SS

### 7. 数据展示规范

#### ✅ 正确做法
- 将查询结果转换为 Markdown 表格
- 标注品种类型（股票/指数/期货）
- 提供数据解读和分析
- 使用百分比格式展示涨跌
- 标注涨跌方向（+/-）

#### ❌ 错误做法
- 直接输出原始 JSON
- 不区分品种类型
- 将数据分析逻辑写成独立脚本
- 创建临时文件处理数据

### 8. 常见分析场景模板

#### 场景1：大盘指数监控
```
分析步骤：
1. 查询主要指数（上证指数、沪深300、创业板指）
2. 计算涨跌幅对比
3. 分析成交额变化
4. 观察指数间联动
5. 总结：整体市场强弱
```

#### 场景2：期货品种追踪
```
分析步骤：
1. 查询黄金、原油、白银等期货
2. 计算各品种涨跌幅
3. 对比相关品种走势
4. 分析联动关系
5. 总结：商品市场整体氛围
```

#### 场景3：批量股票监控
```
分析步骤：
1. 查询多只股票实时行情
2. 按涨跌幅排序
3. 找出领涨领跌股票
4. 分析成交量变化
5. 总结：市场热点和人气
```

#### 场景4：跨品种对比
```
分析步骤：
1. 查询股票、期货等多品种
2. 对比各类资产表现
3. 分析资金流向线索
4. 判断市场风格
5. 总结：大类资产配置思路
```

### 9. 数据质量判断

#### 数据完整性检查
- 检查是否返回了所有请求的字段
- 验证数据条数是否符合请求数量
- 检查关键字段是否为空（最新价、涨跌幅）

#### 数据合理性检查
- 涨跌幅是否在合理范围内（无涨跌停限制）
- 成交量是否为正数
- 价格是否为正数
- 时间戳是否合理

#### 数据时效性检查
- 最新数据时间是否为当前交易时间
- 注意数据延迟（实时行情可能有轻微延迟）
- 非交易时间显示的是最后收盘数据

### 10. 错误处理

#### 常见错误及处理

**错误1：品种代码不存在**
```
症状：返回空数据或无法解析
处理：
1. 确认品种代码格式正确
2. 检查代码是否为支持的格式（sh/sz/hf开头）
3. 确认品种类型（股票/指数/期货）
```

**错误2：请求超时**
```
症状：连接超时或无响应
处理：
1. 检查网络连接
2. 减少同时查询的品种数量
3. 稍后重试
```

**错误3：数据格式异常**
```
症状：返回数据格式不符合预期
处理：
1. 使用 --detail 查看完整数据
2. 使用 --json 获取结构化数据
3. 检查单个品种是否有问题
```

### 11. 性能优化建议

#### 批量查询优化
- 优先使用批量查询（逗号分隔多个代码）
- 避免逐个查询多个品种
- 合理设置输出格式（简单模式更快）

#### 数据策略
- 实时行情按需查询，避免频繁刷新
- 批量监控使用简单模式
- 详细分析使用 detail 模式

#### 并发查询
- 可同时发起多个不同类型的查询
- 例如：同时查询股票和期货
- 注意总查询数量控制

### 12. 合规使用

#### 数据使用规范
- 所有数据仅供展示和分析
- 不得将数据用于商业营利
- 不得对数据进行二次分发
- 尊重数据版权和来源标注

#### 风险提示
- 市场有风险，投资需谨慎
- 数据仅供参考，不构成投资建议
- 投资决策需结合自身判断
- 必要时咨询专业人士
